## 0. שתי ישויות חדשות — קרא קודם

### א. `shows` — תוכנית (כותר)

**"ענבל מצנע" היא תוכנית, לא הפקה.** כל הקלטה היא פרק שלה.
כרגע `podcast_name` הוא טקסט חופשי — ולכן אין לאן לתלות כינויים, אין רווחיות לפי כותר, ואין זיהוי מהיומן.

```sql
shows:
  id, name,                       -- "ענבל מצנע"
  client_id,                      -- מי משלם
  aliases text[],                 -- ["מצנע", "ענבל מצנע", "סופרנואות", "דברים שלמדתי מנשים מצליחות"]
  default_rate numeric,           -- מחיר לפרק
  default_editor_id,              -- עורך קבוע
  default_studio,
  active boolean default true,
  color text                      -- לצביעה ביומן
```

`productions.show_id` מחליף את `podcast_name`.

**זיהוי מהיומן:**
```
אירוע ביומן: "מצנע 14:00"
  → חיפוש ב-aliases (case-insensitive, ללא ניקוד, ללא רווחים כפולים)
  → מצא: ענבל מצנע
  → צור הפקה: show_id=ענבל מצנע, client=אוטומטי, rate=אוטומטי, עורך=אוטומטי
```

⚠️ **חוק ברזל:** שם שלא תואם לאף alias → **המערכת לא ממציאה תוכנית.**
היא יוצרת הפקה במצב `לא_מזוהה` ומרימה דגל: 🟡 *"אירוע ביומן לא זוהה: 'מצנע2'. לאיזו תוכנית לשייך?"*
בלחיצה — אני משייך, **והמערכת לומדת: מוסיפה את המילה כ-alias.**

### ב. `on_hold` — דגל, לא סטטוס

**אם "און הולד" יהיה סטטוס — נאבד את המידע איפה ההפקה הייתה כשעצרה.**

```sql
productions:
  + on_hold boolean default false
  + on_hold_reason text
  + on_hold_since timestamptz
  + on_hold_by uuid
```

הפקה יכולה להיות **"בעריכה + מוקפאת"**. כשמשחררים — היא חוזרת בדיוק לאן שהייתה.
מוקפאת = לא מופיעה בלוח היומי, **אבל כן מופיעה ברדאר** אם היא מוקפאת מעל 14 יום.

**זה סוגר את הלולאה בלי אף מגע ידני.** הלקוח לא צריך חשבון. הלינק פג תוקף.
